Finally done?

Finally, this revision of the site redesign (let's call it 2.0) has been completed. Envisioned from the ground up, the new template boasts powerful multi-device support and stunningly simple site navigation.

From this site's very beginning, I was using Github Pages to render exercises that we had done during my university's "programming basics" class, which covered the basic characteristics of HTML and CSS, as well as a relatively lengthy introduction to Javascript. As future electrical engineers, this was a great stepping stone to C, which we are now using to program Arduino boards, but I digress. Ultimately, this class led to the creation of the "ugly sudoku", a highly dysfunctional batch of Javascript-generated interactive tables and numbers that doesn't really serve a purpose beyond "just being there".

The mess that is the sudoku project.

After a while, I started using the Github repository as public storage for the university's C/microcontroller lectures and lessons, and the notes have already garnered a positive response from some of my classmates, but the site had largely remained unchanged; crude HTML seemed to rule this unorganized waste.

The first version of the site was crude, to put it lightly.

Shortly thereafter, I did some experimentation with a two-column design and flexbox, but I eventually encountered errors when trying to adapt the new page to multiple devices. And I decided then that I needed something simple but functional, or risk looking like a fool with my bare HTML. So I went to the drawing board. Quite literally, in fact.

A sketch of what I thought the site would look like.

And now we're here. Of course, the page isn't like the drawing imagines it, but the result is aimed even more towards my original goals: A no-frills adaptable frame coupled with high-performing and low-bandwidth content. All this using only HTML and CSS, with a dash of Javascript.

From this point on, all I can say is, I hope you enjoy the site. No matter your level of expertise, you are free to help improve it on Github, fork it, or just report bugs and other strange phenomena and artifacts through the bug tracker. I wish you all the best, and I hope to see you again soon.
